# Clarify — 命题分解与决策澄清系统

> 维特根斯坦命题分解 + 苏格拉底反诘法 + 波兰尼默会知识提取

一个基于三套哲学系统的 AI Skill，帮你把模糊的想法变成精准的指令或清晰的决策。

> **Fork 声明**：本项目 Fork 自原作者[riiiku](https://github.com/riiiku) ，由[Stanley-MD](https://github.com/Stanley-MD) 在 Claude 上进行优化与扩展。

## 核心理念

人类表达的困境可以分为三类：

1. **没想清楚的** — 想法模糊、表述混乱
2. **想了但没说出来的** — 已有直觉但无法表达
3. **说不出来的** — 默会知识、只可意会不可言传

Clarify 用三层哲学架构覆盖这三种困境：

```
外层 · 维特根斯坦（扫描）
  → 发现哪里模糊，把复合表述拆成原子单元

中层 · 苏格拉底（挖掘）
  → 用追问把已有但未表达的想法挖出来

内层 · 波兰尼（感应）
  → 处理追问后仍说不清的默会知识
  → 示范法 / 反面法 / 行为提取法
```

## 三种模式

### Prompt 模式 — 模糊任务 → 精准指令

把模糊想法拆解成精准指令。适用于你想让 AI 帮你做一件事，但说不清楚到底要什么。

**触发**：`/clarify prompt` 或 `/clarify`（自动判断）

**SOP 流程**：

1. 命题分类（F/D/Q）— 事实型 / 决策型 / 探索型
2. 模糊词拆解 — 识别并拆解模糊表述
3. 苏格拉底三问 — 追问到底要什么
4. 波兰尼兜底 — 处理说不清的部分
5. 组装结构化指令 — 输出精准 Prompt

**示例**：

```
用户：帮我写个好看的登录页面
Clarify：[经过分解与追问后输出]
→ 生成一个现代风格的登录页面，要求：
  - 深色渐变背景（#1a1a2e → #16213e）
  - 圆角卡片式表单容器，带微妙阴影
  - 输入框聚焦时有蓝色光晕效果
  - 包含"记住我"复选框和"忘记密码"链接
  - 响应式设计，移动端适配
```

### Decision 模式 — 难以表达的困境 → 浮现答案

把说不清楚的困扰、纠结、冲动问清楚。Claude 不替你决定，只帮你挖出你自己的答案。

**触发**：`/clarify decision`

**SOP 流程**：

1. 收集场景 — 了解困境的全貌
2. 语言澄清 — 拆解模糊表述
3. 递进追问 — 苏格拉底式追问
4. 波兰尼兜底 — 处理默会部分
5. 反馈结论 — 帮你看到自己的答案
6. 行动锚点 — 转化为可执行的下一步

**示例**：

```
用户：我想辞职，但又觉得不太对
Clarify：[经过多轮追问后]
→ 你真正纠结的不是"辞不辞职"，而是：
  - 你害怕的不是留下，而是错过外面的可能性
  - 你担心的不是离开，而是离开后证明自己选错了
  → 你的答案已经浮现了，你真正需要的是一个"可逆的尝试"
```

### Fragment 模式 — 混乱思绪 → 捕获原始素材

把碎片化的想法、灵感、感受整理成可用的素材。适用于思绪混乱、灵感闪现、情绪涌动的时刻。

**触发**：`/clarify fragment`

**SOP 流程**：

1. 接收碎片 — 不加评判地接收所有素材
2. 分类整理 — 按主题/情绪/逻辑分组
3. 识别模式 — 发现隐藏的联系与主线
4. 结构化输出 — 转化为可用的知识素材

**示例**：

```
用户：最近总觉得哪里不对，工作没意思，生活也无聊，好像什么都提不起劲...
Clarify：[整理后输出]
→ 你的碎片可以归纳为三个层面：
  - 意义层：工作缺乏价值感
  - 连接层：生活缺少深度关系
  - 活力层：整体能量水平偏低
  → 建议从"活力层"入手，因为能量恢复后，其他两层才有力气处理
```

## 全局约束

Clarify 遵循以下行为准则：

- **零废话** — 不说客套话，不重复用户的话，不写无意义的过渡句
- **不鹦鹉学舌** — 不机械复述用户的表述
- **不说陈词滥调** — 避免"这取决于"、"每个人的情况不同"等万能句式
- **Decision 模式不给未经请求的建议** — 除非用户明确要求，否则不主动给出建议

## 哲学基础

| 哲学家 | 核心思想 | 在系统中的角色 |
|--------|---------|---------------|
| **维特根斯坦** | 语言是思想的边界，说不清=没想清 | 扫描模糊点 |
| **苏格拉底** | 答案在你心里，需要被正确地问出来 | 追问到基岩 |
| **波兰尼** | We can know more than we can tell | 提取默会知识 |

三层哲学家的关系像一张太极图：

- 维特根斯坦告诉你：**想不清楚就别开口**
- 苏格拉底告诉你：**大部分时候你想清楚了，只是没被问到**
- 波兰尼告诉你：**有些东西你怎么问都问不出来，得换一种方式**

三层合在一起，覆盖了人类表达的全部困境。

## 安装与使用

### 前置要求

- [Claude Code](https://claude.ai/claude-code) 或兼容的 AI 编程助手

### 安装步骤

将 `SKILL.md` 文件放入 Claude 的 Skills 目录：

```bash
# 创建技能目录
mkdir -p ~/.claude/skills/clarify/

# 复制技能文件
cp SKILL.md ~/.claude/skills/clarify/
```

### 触发命令

| 命令 | 说明 |
|------|------|
| `/clarify` | 自动判断模式（根据输入内容智能选择） |
| `/clarify prompt` | 强制 Prompt 模式 |
| `/clarify decision` | 强制 Decision 模式 |
| `/clarify fragment` | 强制 Fragment 模式 |
